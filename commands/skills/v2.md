# v2

## When to use this skill

Load this skill when the user's request involves any of these operations on v2:

- **find-suggest** — Find Suggest
- **find-venues** — Venue Search
- **get-classification-details** — Get Classification Details
- **get-event-details** — Get Event Details
- **get-event-images** — Get Event Images
- **get-genre-details** — Get Genre Details
- **get-segment-details** — Get Segment Details
- **get-subgenre-details** — Get Sub-Genre Details
- **get-venue-details** — Get Venue Details
- **search-classifications** — Classification Search

## Authentication

This API requires authentication. Configure once with `discovery-api configure`, or set the env var directly:

- `DISCOVERY_API_API_KEY` (apiKey)

## Commands

### `discovery-api v2 find-suggest`

Find Suggest

- **HTTP**: `GET /discovery/v2/suggest`
- **Safety**: safe=true, idempotent=true, reversible=true, impact=low
- **Auth required**: true

Flags:

- `--keyword` `string` — Keyword to search on
- `--source` `string` — Filter entities by its source name [one of: `ticketmaster`, ` universe`, ` frontgate`, ` tmr`]
- `--latlong` `string` — Filter events by latitude and longitude, this filter is deprecated and maybe removed in a future release, please use geoPoint instead
- `--radius` `string` — Radius of the area in which we want to search for events.
- `--unit` `string` — Unit of the radius [one of: `miles`, `km`]
- `--size` `string` — Size of every entity returned in the response
- `--include-fuzzy` `string` — yes, to include fuzzy matches in the search. This has performance impact. [one of: `yes`, ` no`]
- `--client-visibility` `string` — Filter events to clientName
- `--country-code` `string` — Filter suggestions by country code
- `--include-tba` `string` — True, to include events with date to be announce (TBA) [one of: `yes`, ` no`, ` only`]
- `--include-tbd` `string` — True, to include event with a date to be defined (TBD) [one of: `yes`, ` no`, ` only`]
- `--segment-id` `string` — Filter suggestions by segment id
- `--geo-point` `string` — filter events by geoHash
- `--locale` `string` — The locale in ISO code format. Multiple comma-separated values can be provided. When omitting the country part of the code (e.g. only 'en' or 'fr') then the first matching locale is used. When using a '*' it matches all locales. '*' can only be used at the end (e.g. 'en-us,en,*') 
- `--include-licensed-content` `string` — Yes if you want to display licensed content [one of: `yes`, ` no`]
- `--include-spellcheck` `string` — yes, to include spell check suggestions in the response. [one of: `yes`, ` no`]

Example:
```bash
discovery-api v2 find-suggest
```

## Workflows

### Branch on exit code instead of parsing stderr
```bash
discovery-api v2 <command> [flags]
case $? in
  0) : success ;;
  2) echo "fix credentials" ;;
  3) echo "not found, skip" ;;
  4) echo "validation error, fix flags" ;;
  5) sleep 10 ; retry ;;
  6) echo "server down, abort" ;;
esac
```

## Gotchas
- Pass `--schema` (or `--help` in agent mode via `CLAUDE_CODE=1`) to any command for a JSON schema of its inputs and outputs.
- Errors go to stderr as a single JSON line; stdout stays clean for piping. Branch on `$?` instead of parsing stderr.
- See `discovery-api skills get _global` for cross-cutting flags (`--output-format`, `--jq`, `--debug`) and exit code reference.
